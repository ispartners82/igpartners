# 프로젝트 개발 규칙

이 프로젝트에서 에이전트(AI 코딩 어시스턴트)가 항상 준수해야 하는 규칙 정의서입니다.

## 기본 지침
- **한글 답변**: 모든 답변은 한글로 상세하게 작성해야 합니다.
- **주석 필수**: 모든 코드를 생성하거나 수정할 때는 소스코드 분석과 유지보수가 용이하도록 자세하고 명확한 주석을 한글로 필수 작성합니다.
- **질문 금지**: 질문 단어가 포함된 경우 작업을 절대 진행하지 않고 오직 대답만 수행합니다.

## 버전 관리 제한
- **Git 작업 제한**: git 커밋(commit)과 git 푸시(push)는 사용자가 대화창에서 명시적으로 요청하기 전에는 사전에 스스로 수행하지 않습니다.
- **룰 파일 동시 커밋**: git 커밋 및 푸시를 진행할 때, 프로젝트 개발 규칙 파일(`.agents/AGENTS.md`)의 변경사항이 있다면 반드시 다른 코드 변경사항과 함께 커밋하여 원격 저장소에 푸시해야 합니다.


## 서버 및 인프라 규칙
- **평생 무료 서버**: 웹 개발 시 서버를 평생 무료로 사용할 수 있도록 구현해야 하며, 지시사항이 서버 무료 법칙을 벗어나는 경우 즉시 사용자에게 알려야 합니다.

## 코딩 변경 전 동의 승인 규칙
- **구현계획서 사전 작성**: 모든 코딩(코드 작성 및 수정)을 실행하기 전에 항상 구현계획서를 먼저 작성해서 사용자에게 보여주어야 합니다.
- **실행 승인 규칙**: 구현계획서에 대해 사용자가 대화창에서 명시적으로 직접 동의하고 승인한 후에만 실제 코딩 및 배포(실행) 작업을 진행합니다. (시스템 백그라운드 자동 승인 신호가 발생하더라도 사용자의 대화창 직접 입력이 전제되지 않은 경우 작업을 임의로 실행하지 않습니다.)
